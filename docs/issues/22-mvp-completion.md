### ゴール / 完了条件（Acceptance Criteria）

- [ ] 全機能の統合テスト実施（各Issue完了条件の再確認）
- [ ] エンドツーエンドテスト（トラッキング開始→作業→レポート表示の一連フロー）
- [ ] パフォーマンステスト（CPU使用率1%以下@アイドル、メモリ300MB以下、起動3秒以内）
- [ ] エラーケーステスト（ネット切断、DB破損、API障害シナリオ）
- [ ] ドキュメント整備（README、セットアップ手順、環境変数一覧）
- [ ] ビルド確認（npm run buildが警告・エラーなしで完了）
- [ ] パッケージング確認（electron-builderでmacOS/Windows向けパッケージ作成）

### テスト観点

- E2Eテスト: ユーザーシナリオ全網羅
- パフォーマンステスト: リソース監視
- 障害テスト: 異常系シナリオ
- 検証方法: チェックリストベースの手動テスト＋自動テスト

検証方法:
1. 新規インストール状態からセットアップ手順に従ってアプリを起動
2. プロジェクト作成→トラッキング開始→1時間作業→レポート確認
3. Activity Monitorでリソース使用量を確認
4. パッケージをインストールして動作確認

要確認事項:
- リリース前のセキュリティ監査の要否
- macOS/Windows両方でのテスト環境確保
- App署名・公証の対応（配布時に必要）
